Setting up a working blog site using Hexo/Nginx/AWS from beginning (2/7)

Posted on 2021-01-09

This is a series of notes (2 out of 7) regarding steps of making this site work. Every step, more or less, is through internet searching, so this article is a list of websites for future reference. Also note that this exercise is for personal practice only, so it only has very limited function and is not intended for larger traffic.

To get a taste, try to install docker and use hexo and/or nginx docker images , which should be working out of box.

For a more traditional LAMP (Linux/Apache/M) website set up, you can refer to other sites

Configure EC2 on AWS

New users can get a free Tier on AWS. Here are the steps to make my tiny Ubuntu server up and running:

  1. log into Amazon AWS with your credentials, select log in as root user;

  2. Click on Services on up-left corner then select EC2;

  3. You are now on a page shows all resources, and in the middle of the page, click the orange box where is says “Launch instance”;

  4. Follow the instructions to setup your Amazon Machine Image (AMI), I used “Ubuntu Server 20.04 LTS (HVM), SSD Volume Type” (64-bit, X86);

  5. Select the t2.micro type (Free tier eligible), then click “Next: Configure Instance Details”;

  6. Default values work okay in step 3 and 4, in step 5, you can set a key-value pair, I used key=ttf1 and value = halfdayfree.com, for example;

  7. Next we need configure the security group, add HTTPS and HTTP to the rule list, then click Review and Launch;

  8. Create a new key pair, give it a name (halfdayfree, for example), the download and save the private key file, click Launch Instances;

  9. View your instance and you can check all relative information regarding your virtual server. Select your instance in check box and then click Connect on top of the page;

  10. click “connect” on low-right position and you should be able see a black linux command line interface.

Congratulations! You now have a working tiny linux server, with a public ip address.

In the next post, we will use the private key file downloaded so you can access from any SSH client, rather than through AWS page each time!

Happy coding!
